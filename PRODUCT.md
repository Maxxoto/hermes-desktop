# Hermes Desktop

## Product
Native macOS desktop client for Hermes Agent — connects to a self-hosted Hermes Gateway API over LAN/Tailscale. Chat interface, session management, connection configuration.

## Register
Product (app UI) — tool interface, not marketing.

## Target Users
- **Primary:** Self-hosted Hermes Agent operators running macOS
- **Secondary:** Developers building on the Hermes platform
- **Context:** Power users who run a homelab (Raspberry Pi / NUC) with Hermes Gateway, want a native desktop experience instead of web UI

## Purpose
Replace browser-based Hermes Gateway interaction with a native macOS app — faster, cleaner, better system integration (menu bar, keyboard shortcuts, vibrancy).

## Desired Outcome
A macOS-native chat app that feels like Raycast/Linear — fast, minimal, glass-morphism, keyboard-first.

## Brand Personality
Minimal, precise, warm. Not cold/enterprise. Think: a well-crafted developer tool, not a SaaS dashboard.

## Anti-References
- Heavy SaaS dashboards (complex nav, metric cards, gradient headers)
- AI slop: Inter everywhere, purple gradients, nested cards, side-tab borders, gradient text
- Generic chat apps (Slack/Teams complexity)

## Strategic Design Principles
1. macOS-native minimalism — Apple HIG, vibrancy, system fonts
2. Keyboard-first — Cmd+K palette, shortcuts for everything
3. Single-purpose screens — chat is chat, config is config
4. Glass + depth — subtle blur, layered surfaces, not flat
